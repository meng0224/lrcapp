# LrcApp Figma 功能規格提示詞

本文件提供一段可直接貼進 Figma 或其他規格整理工具的單一提示詞。內容只保留目前 repo 的真實功能邏輯、流程限制，以及頁面上必須存在的功能區塊與開關/卡片清單。

## 使用方式

- 直接複製下方「主提示詞」整段使用。
- 若需要補充限制，優先補在提示詞最後，不要改動中段的功能邏輯。
- 這份提示詞的用途是整理功能規格，不是產出設計稿要求。

## 主提示詞

```text
請根據以下資訊，整理一份 LrcApp 的單頁 Android 工具功能規格。只需要描述功能邏輯、流程限制、必備功能區塊、開關、卡片、操作入口與狀態條件。

產品名稱是 LrcApp，功能是把常見字幕格式批次轉換成 LRC 歌詞格式。

這個產品目前的核心流程固定是：
設定輸出位置 -> 選擇文件或資料夾 -> 驗證 -> 開始轉換 -> 自動保存

請嚴格遵守以下不可改動的產品邏輯，不要新增功能，不要改變流程：

1. 這是一個 Android 單頁工具，核心任務都在同一個主頁完成。
2. 支援輸入格式：VTT、ASS、SSA、SRT、STR、SMI、SUB。輸出格式固定為 LRC。
3. 一般模式下，使用者透過系統文件選擇器一次選擇多個字幕檔。
4. 若開啟「遞迴匯入子資料夾字幕」，選取入口會改為選擇資料夾，並遍歷子資料夾匯入字幕檔。
5. 驗證規則固定存在：不支援格式要標示為無效，單檔超過 10MB 也要標示為無效。
6. 有三種輸出模式：
   - 預設應用下載目錄
   - 自訂 SAF 授權目錄
   - 輸出到原文件目錄
7. 自訂 SAF 授權目錄與輸出到原文件目錄互斥，不能同時啟用。
8. 若啟用輸出到原文件目錄，第一次寫入某個來源資料夾時需要授權；若是遞迴匯入，會先對匯入根目錄授權，後續保存沿用同一授權並重建原始子資料夾結構。
9. 一般模式下，第二次選檔會覆蓋舊列表；原文件目錄模式下，第二次選檔會追加到現有列表；遞迴匯入也會追加並去除重複 Uri。
10. 遞迴匯入時，不支援格式或超過 10MB 的檔案會直接略過，不加入列表。
11. 轉換完成後會自動保存所有成功檔案，不提供單檔下載、ZIP 打包、歌詞預覽、雲端同步或批次設定面板。
12. 若保存時失敗，例如 SAF provider 改寫副檔名導致結果不是有效的 .lrc，不能誤報成功，必須明確歸類為保存失敗。
13. smart naming 與 time precision 目前固定啟用，但沒有可操作的開關，不要把它們列成可設定項。

請整理這個主頁面必須包含的功能區塊與控制項，只描述它們承載的功能，不要描述樣式：

- App bar：用來顯示產品名稱與工作台性質。
- 儲存位置卡片：顯示目前輸出模式、目前輸出位置，以及和輸出模式相關的操作入口。
- 自訂輸出目錄相關操作：至少包含選擇目錄與清除目錄。
- 原文件目錄開關：控制是否輸出回來源資料夾。
- 遞迴匯入開關：控制是否改成資料夾匯入模式。
- 授權提示區：當來源目錄或匯入根目錄需要重新授權時，用來顯示目前等待處理的授權事項，並提供授權入口。
- 文件列表區：顯示目前加入的字幕檔、各檔案狀態、錯誤訊息與成功輸出資訊。
- 進度卡片：在轉換中顯示整體進度、目前處理狀態與處理數量。
- 底部主操作區：承載主操作與次操作。
- 主操作按鈕：依情境執行選擇文件、選擇資料夾或開始轉換。
- 次操作按鈕：至少包含重新選擇或新增匯入來源，以及清除文件列表。
- 回饋機制：用來回報匯入結果、清除結果、授權需求、保存成功或保存失敗。

請完整列出以下狀態邏輯，只描述功能條件與行為：

1. 預設空列表狀態。
2. 已設定自訂輸出目錄。
3. 已開啟輸出到原文件目錄。
4. 已開啟遞迴匯入子資料夾字幕。
5. 文件列表同時存在待處理、無效、處理中、成功、失敗。
6. 轉換進行中。
7. 等待來源目錄或匯入根目錄授權。
8. 部分保存成功、部分保存失敗。
9. 沒有可轉換文件，因此不可開始轉換。

文件列表中的單筆檔案至少要能表達以下資訊：

- 檔名
- 檔案大小
- 檔案格式
- 目前狀態
- 成功時的輸出檔名
- 失敗或無效時的錯誤訊息

請把檔案狀態明確列為：

- 待處理
- 無效
- 處理中
- 成功
- 失敗

請最後輸出一份純功能規格，至少包含：

1. 產品目的
2. 核心流程
3. 不可改動的功能邏輯
4. 主頁必備功能區塊
5. 必要開關、卡片、按鈕與操作入口
6. 檔案列表需要承載的資料
7. 關鍵狀態條件與對應行為
8. 明確排除的非目標功能

請避免以下錯誤：

- 不要新增不存在的功能
- 不要把 smart naming 或 time precision 寫成可切換設定
- 不要把這份內容寫成設計稿需求，而要寫成功能規格
```
